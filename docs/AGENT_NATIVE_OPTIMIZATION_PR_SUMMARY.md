# Agent-Native Optimization PR Summary

## Why

Git.Top already had a strong agent-native foundation: REST, MCP, GRP, OpenAPI, `llms.txt`, `llms-full.txt`, Agent Map, Trust Gate, quality reports, and evaluation scripts.

This change set makes those surfaces easier for external agents and maintenance agents to trust:

- Better machine-readable contracts.
- More uniform evidence fields.
- Public benchmark visibility.
- Clearer MCP behavior examples.
- A shorter root guide for future coding agents.

## What Changed

### Maintenance Agent Onboarding

- Added `AGENTS.md` as the short safe entry point for coding agents.
- Linked the guide from README, REST API docs, and MCP docs.
- Added `docs/AGENT_NATIVE_ASSESSMENT_AND_OPTIMIZATION_PLAN.md`.
- Added `docs/AGENT_NATIVE_OPTIMIZATION_IMPLEMENTATION_REPORT.md`.

### REST and OpenAPI Contracts

- Expanded OpenAPI response schemas for the public agent-facing API surface.
- Added shared schemas for metadata, errors, project cards, quality confidence, Trust Gate checks, recommendations, and benchmark responses.
- Added validation so public `GET` endpoints do not regress to schema-less OpenAPI responses.
- Added `POST /mcp` JSON-RPC request, success response, and error response schemas, including `result.content[0].text` parsing and strict D1 error code guidance.
- Added `POST /api/grp/query` request schema coverage and a generic validation guard for public `POST` request/response schemas.
- Fixed post-review contract drift: MCP OpenAPI now covers `initialize` and `notifications/initialized`, GRP `explanation` is documented as an array, and MCP discovery separates structured POST endpoints from read-only GET endpoints.

### Evidence Normalization

- Added normalized evidence packages across project lookup, recommendations, score explanations, alternatives, graph, and GRP responses.
- Standardized key fields where possible: `evidence`, `caveats`, `confidence_reason`, `source_fields`, and `last_verified_at`.
- Added explanation eval checks for normalized evidence.

### Public Trust Benchmark

- Added `/benchmark` and `/api/benchmark`.
- Added MCP tool `get_public_benchmark`.
- Linked benchmark surfaces from trust, quality, docs, README, sitemap, Agent Map, OpenAPI, and LLM discovery.
- Added baseline validation to keep public benchmark constants aligned with generated eval reports.

### MCP Integration Hardening

- Added `docs/MCP_TOOL_BEHAVIOR_EXAMPLES.md`.
- Added `docs/SDK_EXAMPLES.md`.
- Added `agent_api.response_contract` to MCP discovery.
- Documented `POST /mcp` in OpenAPI as a JSON-RPC integration surface rather than a prose-only endpoint.
- Expanded strict `require_d1` validation for REST and MCP high-confidence flows.
- Added OpenAPI validation to prevent public structured `POST` endpoints from regressing to prose-only request or response contracts.
- Added validation for MCP discovery endpoint grouping and MCP/GRP schema-runtime alignment.
